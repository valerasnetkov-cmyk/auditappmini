## Production demo card 404 hotfix

- Причина риска: production-база могла содержать уже созданную demo-компанию,
  но без актуального обязательного набора техники после предыдущего seed.
- Добавлена проверка готовности публичного demo-набора при `/api/auth/demo`:
  backend проверяет demo-компанию, demo-пользователя, количество техники и
  наличие `demo-vehicle-01`; при неполном наборе запускает безопасный
  `provisionPublicDemo`.
- Ожидаемый эффект: после повторного входа в demo карточка
  `/vehicles/demo-vehicle-01` получает данные из API вместо `HTTP 404`.
- Дополнительный frontend hotfix: запрос `/vehicles/:id/photo-options` больше
  не переводит всю карточку техники в общий error-state при 404. Если
  production API ещё не отдаёт этот optional endpoint или данные фото
  недоступны, карточка остаётся открытой, а список выбора фото отображается
  пустым.

## Frontend performance P1

- Яндекс Метрика в root layout переведена с `afterInteractive` на
  `lazyOnload`; фактическая загрузка `tag.js` и `ym(..., init)` дополнительно
  откладываются на 7 секунд после `lazyOnload`, а затем запускаются через
  `requestIdleCallback` с fallback, чтобы внешний скрипт не участвовал в
  первичной отрисовке и не влиял на FCP/LCP.
- Подключён `@next/bundle-analyzer`; анализ запускается стандартным флагом
  `ANALYZE=true npm run build` из каталога `web`.
- Из-за Turbopack в Next 16 добавлен wrapper `web/scripts/build.mjs`: обычная
  сборка остаётся `next build`, а при `ANALYZE=true` автоматически используется
  `next build --webpack`, чтобы analyzer генерировал отчёты.
- Статический импорт `exceljs` убран из hook страницы настроек: парсер Excel
  загружается динамически только при запуске импорта техники из файла.
