# Accident Inspection: Place and Time

## Цель

Для осмотра ДТП обязательно фиксируются место и время ДТП. Эти данные нужны для доказательной карточки осмотра и не заменяются автоматически временем создания осмотра или геоданными фото.

---

## Обязательные данные ДТП

Для `inspection.type = accident` перед завершением осмотра должны быть заполнены:

```txt
accident_occurred_at      -> дата и время ДТП
accident_place_text       -> место ДТП текстом: адрес, трасса, объект, парковка или ориентир
accident_latitude         -> координата места ДТП, если доступна
accident_longitude        -> координата места ДТП, если доступна
accident_location_source  -> источник места: gps / manual / corrected
accident_time_source      -> источник времени: manual / device / corrected
```

Минимально обязательные поля:

```txt
accident_occurred_at
accident_place_text
```

Координаты желательно фиксировать всегда, если устройство может получить геолокацию. Если координаты недоступны, инспектор всё равно обязан указать место ДТП текстом.

---

## Правила

- Время ДТП вводится или подтверждается инспектором.
- Место ДТП вводится или подтверждается инспектором.
- Время начала осмотра не считается временем ДТП.
- Геоданные фото не считаются единственным источником места ДТП.
- Если приложение определило геолокацию автоматически, инспектор должен увидеть её и подтвердить или исправить место текстом.
- Для ДТП нельзя завершить осмотр без места и времени ДТП.

## UX mobile

В сценарии ДТП перед фото повреждений показывается блок:

```txt
Данные ДТП

Дата и время ДТП
[ 02.05.2026 14:30 ]

Место ДТП
[ г. Москва, парковка ..., ориентир ... ]

Координаты
[ Определить автоматически ]

[ Подтвердить данные ДТП ]
```

Если приложение получило координаты, инспектор всё равно заполняет или подтверждает текстовое место.

## Web

В карточке ДТП-осмотра отображаются:

```txt
Дата и время ДТП
Место ДТП
Координаты ДТП, если есть
Источник места
Источник времени
Дата и время осмотра
Инспектор
```

Важно визуально разделять:

```txt
время ДТП != время осмотра
место ДТП != место загрузки/просмотра фото
```

---

## Backend validation

Перед завершением `accident`-осмотра backend проверяет:

```txt
inspection.type = accident
accident_occurred_at is not null
accident_place_text is not empty
```

Если данные отсутствуют, возвращается ошибка:

```json
{
  "error": "inspection_validation_failed",
  "missingData": ["accidentOccurredAt", "accidentPlace"]
}
```

---

## Data model

Поля можно хранить в таблице `inspections`, потому что ДТП является типом осмотра:

```txt
inspections.accident_occurred_at
inspections.accident_place_text
inspections.accident_latitude
inspections.accident_longitude
inspections.accident_location_source
inspections.accident_time_source
inspections.accident_confirmed_by
inspections.accident_confirmed_at
```

---

## Что не делать

- Не подставлять время создания осмотра как время ДТП без подтверждения инспектора.
- Не считать GPS фото полноценным описанием места ДТП.
- Не завершать ДТП-осмотр без места и времени ДТП.
- Не смешивать в интерфейсе дату ДТП и дату осмотра.
