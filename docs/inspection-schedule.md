# План-график осмотров

P1-сценарий рассчитывает сроки быстрых и плановых осмотров для каждой единицы
техники. В расчёт входят только технически завершённые осмотры.

## Настройки

На уровне компании хранятся:

- `default_quick_inspection_interval_days`, по умолчанию 7;
- `default_planned_inspection_interval_days`, по умолчанию 30.

На уровне техники доступны nullable-переопределения:

- `quick_inspection_interval_days`;
- `planned_inspection_interval_days`.

Пустое переопределение означает наследование настройки компании. Интервалы
принимают целые значения от 1 до 3650 дней.

## Статусы

- `inspection_actual` — следующий срок позже порога предупреждения;
- `inspection_due_soon` — срок наступит в пределах
  `notification_days_before`;
- `inspection_overdue` — срок уже прошёл;
- `never_inspected` — завершённого осмотра соответствующего типа ещё нет.

API техники возвращает `inspection_schedule` с отдельными блоками `quick` и
`planned`, а также общий статус с приоритетом наиболее опасного состояния.
`GET /api/vehicles` принимает фильтр `inspection_status`.

## Web и уведомления

Список техники показывает общий статус графика и фильтруется по нему. Карточка
техники показывает последний и следующий срок отдельно для быстрого и
планового осмотра. Интервалы компании редактируются в настройках, а
индивидуальные интервалы — в форме техники.

Dashboard-напоминания используют тот же расчёт и создают отдельные строки для
быстрого и планового осмотра.
