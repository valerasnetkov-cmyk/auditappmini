# Web Changes

## Цель

Web-интерфейс нужен руководителю и администратору для просмотра техники, истории осмотров, фото, дефектов, пробега, ДТП-фиксаций и настроек компании.

---

## Основные страницы MVP

```txt
/dashboard
/vehicles
/vehicles/:id
/inspections
/inspections/:id
/settings/company
/settings/users
```

---

## Список техники

Колонки:

- номер;
- название;
- статус;
- последний осмотр;
- тип последнего осмотра;
- последний километраж;
- наличие дефектов;
- наличие ДТП-осмотров;
- дата обновления.

Фильтры:

- поиск по номеру;
- статус;
- есть дефекты;
- тип последнего осмотра;
- дата осмотра;
- инспектор.

---

## Карточка техники

Блоки:

- номер машины;
- название / тип техники;
- последний осмотр;
- последний подтверждённый километраж;
- история пробега;
- дефекты;
- история осмотров;
- фото по осмотрам;
- ДТП-фиксации.

---

## Карточка осмотра

Карточка осмотра должна показывать:

- компанию;
- технику;
- номер;
- тип осмотра: быстрый / плановый / ДТП;
- инспектора;
- дату и время начала;
- дату и время завершения;
- геоданные;
- результат осмотра;
- километраж;
- фото одометра;
- источник километража: OCR / ручной ввод / исправлено после OCR;
- чек-лист;
- дефекты;
- галерею обязательных фото.

---

## Галерея фото осмотра

Фото группируются по типу:

```txt
Фронтальная часть
Левый борт
Правый борт
Задняя часть
Общий план
Одометр
Дефекты
ДТП: общий план повреждения
ДТП: крупный план повреждения
Дополнительные фото
```

Если обязательное фото отсутствует, web должен показывать предупреждение.

---

## ДТП-осмотр

Для осмотра ДТП в web должны быть видны:

- общий план автомобиля;
- общий план повреждённой зоны;
- крупный план каждого повреждённого участка;
- описание повреждений;
- место ДТП;
- дата и время ДТП;
- координаты ДТП, если есть;
- связанный дефект или запись повреждения;
- километраж, если доступен;
- причина отсутствия километража, если он недоступен.

В web нужно визуально отделять дату/время ДТП от даты/времени проведения осмотра.

---

## Одометр и пробег

В карточке осмотра отображается:

```txt
Пробег: 128450 км
Источник: распознано и подтверждено инспектором
Фото одометра: открыть
```

В карточке техники отображается последнее подтверждённое значение.

Желательно добавить историю пробега:

| Дата | Осмотр | Пробег | Инспектор | Источник |
|---|---|---:|---|---|

## Настройки компании

Настройки:

- название компании;
- язык по умолчанию;
- единицы измерения пробега: километры / мили;
- требования к фото по типам осмотра;
- шаблоны чек-листов;
- пользователи и роли.

---

## Мультиязычность

Поддерживаются языки:

```txt
ru
en
de
fr
es
```

Переводятся интерфейсные тексты, типы осмотров, типы фото, ошибки и статусы.

---

## Стили CSS

CSS выносится во внешние файлы:

```txt
web/src/styles/
```

Inline-стили в компонентах не используются.

---

## Что не делать в web

- Не показывать данные другой компании.
- Не давать скачать фото без проверки доступа.
- Не редактировать подтверждённый осмотр без audit log.
- Не скрывать факт ручного исправления OCR-значения.
- Не смешивать фото разных типов в одну неструктурированную галерею.

## Тема интерфейса для ПК / Web

Web-приложение должно поддерживать три режима темы:

```txt
system
light
dark
```

Где используется:

- экран входа;
- выбор компании;
- Dashboard;
- список техники;
- карточка техники;
- карточка осмотра;
- галерея фото;
- дефекты;
- настройки компании и пользователя.

В настройках пользователя добавить поле:

```txt
Тема интерфейса: Как в системе / Светлая / Тёмная
```

Технические правила:

- тема применяется через `data-theme` на корневом элементе;
- цвета берутся из CSS variables;
- inline-стили для цветов запрещены;
- переключение темы должно происходить без повторного входа;
- выбор темы сохраняется в профиле пользователя;
- при `system` используется системная тема браузера.

Для SaaS-ready режима у компании может быть `default_theme`, но личная настройка пользователя имеет приоритет.

## Regional web behavior

Web-приложение должно определять tenant по поддомену или выбранной компании и получать регион из tenant registry.

```txt
<tenant>.<project-domain> -> tenant registry -> region_code -> api base
```

Web не должен позволять пользователю вручную менять регион API через URL-параметры. Если tenant относится к `ru`, все запросы идут в RU API.

Если регион недоступен, показывается ошибка:

```txt
Региональный сервис временно недоступен. Попробуйте позже.
```

В интерфейсе владельца/администратора компании можно показывать регион хранения данных только как справочную информацию. Изменять регион через обычные настройки компании нельзя.


## Настройка единиц пробега

В панели администратора компании должна быть настройка:

```txt
/settings/company -> Единицы измерения -> Пробег
```

Варианты:

```txt
Километры (km)
Мили (mi)
```

Изменять могут только `owner` и `admin`.

При изменении показать предупреждение:

```txt
Новые осмотры будут использовать выбранную единицу измерения. Старые записи сохранят исходную единицу.
```

Во всех карточках и таблицах пробег отображается с единицей:

```txt
128450 км
79815 mi
```

## Web: OCR history and diagnostics

В web/admin панели можно добавить просмотр попыток распознавания номера для диагностики качества OCR.

Минимально:

```txt
- дата попытки;
- инспектор;
- raw_text;
- normalized_number;
- confirmed_number;
- confidence;
- provider;
- provider_region;
- статус confirmed / failed.
```

Доступ к этой информации должен быть только у manager/admin/owner внутри своей компании.
