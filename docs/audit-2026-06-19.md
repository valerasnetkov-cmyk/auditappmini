## Production demo availability

- Проверен live endpoint `POST https://auditavto.ru/api/auth/demo` и
  `POST https://api.auditavto.ru/api/auth/demo`: сервер отвечает
  `demo_unavailable`, значит route доступен, но production env не включает
  public demo/provisioning.
- Обновлён `backend/.env.production.example`: для публичного сайта
  `PUBLIC_DEMO_ENABLED=true`, `PUBLIC_DEMO_PASSWORD` должен быть задан сильным
  значением. Production doctor уже блокирует включённое демо без пароля длиной
  минимум 12 символов.
- Backend больше не отвечает `404` для выключенного demo route: используется
  `503 demo_unavailable`, чтобы live console не выглядела как отсутствующий
  endpoint.
- Для исправления на рабочем сервере после pull/restart нужно задать в
  `backend/.env.production` или secret manager:
  `PUBLIC_DEMO_ENABLED=true` и `PUBLIC_DEMO_PASSWORD=<strong value>`.
