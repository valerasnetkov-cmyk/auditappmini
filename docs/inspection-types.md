# Inspection Types

## Назначение

Файл описывает типы осмотров, их назначение и минимальный набор данных.

---

## Перечень типов

В MVP используются три типа осмотра:

```txt
quick    -> быстрый осмотр
planned  -> плановый осмотр
accident -> осмотр ДТП / инцидента
```

---

## Быстрый осмотр

### Назначение

Регулярная быстрая проверка техники. Используется чаще всего.

### Цель

За короткое время зафиксировать состояние автомобиля и сохранить доказательные фото.

### Обязательные данные

- подтверждённый номер автомобиля;
- фото фронтальной части;
- фото левого борта;
- фото правого борта;
- фото задней части;
- общий план;
- фото одометра;
- подтверждённый километраж;
- короткий чек-лист.

### Результат

```txt
ok          -> дефектов не найдено
has_defects -> есть дефекты
```

---

## Плановый осмотр

### Назначение

Проверка по графику или внутреннему регламенту компании.

### Отличие от быстрого

Плановый осмотр использует расширенный чек-лист и может требовать больше общих фото.

### Обязательные данные

- все обязательные фото быстрого осмотра;
- расширенный чек-лист;
- дополнительные общие планы при необходимости;
- фото одометра;
- подтверждённый километраж;
- комментарии по пунктам, где выявлены отклонения.

---

## Осмотр ДТП

### Назначение

Фиксация состояния автомобиля после ДТП, повреждения или инцидента.

### Обязательные данные

- общий план автомобиля;
- фото фронтальной части;
- фото левого борта;
- фото правого борта;
- фото задней части;
- общий план повреждённой зоны;
- крупный план каждого повреждённого участка;
- описание повреждения;
- место ДТП;
- дата и время ДТП;
- координаты ДТП, если доступны;
- фото одометра, если доступно;
- подтверждённый километраж или причина невозможности фиксации.

### Важное правило

Общий план повреждения не заменяет крупный план. Для каждого повреждения нужен отдельный close-up.

Место и время ДТП являются обязательными данными. Время начала осмотра не заменяет время ДТП, а геоданные фото не заменяют текстовое место ДТП.

---

## Статусы осмотра

```txt
draft      -> осмотр создан, но не завершён
in_progress -> осмотр выполняется
completed  -> осмотр завершён
cancelled  -> осмотр отменён
```

---

## Правила завершения

Осмотр нельзя завершить, если:

- не подтверждён номер машины;
- не выбран тип осмотра;
- отсутствуют обязательные фото;
- отсутствует фото одометра для быстрого или планового осмотра;
- километраж не подтверждён и не указан вручную;
- есть ответ `НЕТ` без фото дефекта;
- для ДТП нет крупного плана повреждения;
- для ДТП не указано место ДТП;
- для ДТП не указаны дата и время ДТП;
- отсутствуют дата, время или геоданные.

## Плановый осмотр: расширенные блоки

Плановый осмотр состоит из следующих разделов:

| Раздел | Что проверяется | Фото |
|---|---|---|
| `exterior` | внешние дефекты кузова, стёкол, зеркал, номерного знака | обязательные внешние ракурсы, `defect` при отклонении |
| `undercarriage` | доступные элементы ходовой части, подвески, колёс и шин | `undercarriage` / `defect` при отклонении |
| `brake_system` | видимые элементы тормозной системы, подтёки, предупреждения на панели | `brake_system` / `defect` при отклонении |
| `electrical` | аккумулятор, клеммы, видимая проводка, приборная панель, стеклоочистители | `electrical` / `defect` при отклонении |
| `lighting` | ближний/дальний свет, габариты, стоп-сигналы, поворотники, аварийка, задний ход, подсветка номера | `lighting` / `defect` при отклонении |
| `odometer` | фото одометра и подтверждённый километраж | `odometer` обязательно |

Детальные фото технических узлов не должны быть обязательными для каждого пункта по умолчанию. Они обязательны при ответе `НЕТ`, видимом дефекте или требовании регламента компании.

Если узел недоступен без разборки, инспектор не разбирает автомобиль, а отмечает пункт как недоступный или оставляет комментарий.
