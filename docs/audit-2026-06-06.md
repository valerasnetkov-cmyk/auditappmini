# Audit 2026-06-06

Ежедневная сводка изменений за 2026-06-06. Файл содержит только изменения,
зафиксированные в этот день.

## SaaS Tariff and Billing Policy

- Реализована тарифная сетка `pilot` / `standard` / `enterprise` с ценами,
  лимитами техники, пользователей, осмотров, хранилища, OCR и feature flags.
- Существующая модель `company_subscriptions` / `company_payments` сохранена
  для совместимости; добавлены `company_billing`, `company_billing_events` и
  `company_ocr_usage`.
- Добавлен resolver тарифной политики с приоритетом индивидуальных
  `company_limits` над базовым `plans`.
- Backend блокирует превышение лимитов техники, пользователей и месячных
  осмотров, отключённые OCR/ДТП/аналитику/экспорт и операции записи для
  `suspended`/`archived`.
- `/api/company/usage` расширен данными тарифа, оплаты, осмотров за месяц,
  фото-хранилища, OCR и доступных модулей; кабинет компании отображает эти
  значения и предупреждения.
- Для resource admin добавлены `/api/admin/saas/plans` и billing/payment/limits
  endpoints с обязательной историей событий.
- Добавлен repeat-safe `backend/scripts/billing-status-job.mjs` и команда
  `npm --prefix backend run billing:check`.
- Добавлен `smoke:billing-policy`; целевой smoke, unit tests, web lint и web
  build проходят.
- Финальный `npm run verify:launch` прошёл полностью: 41 backend unit test,
  весь smoke-набор, web build, mobile verify, 9 Chromium E2E, launch doctors и
  dependency audits (0 vulnerabilities).

## AuditAvto Sales Landing

- Публичный лендинг `/` обновлён по `auditavto_landing_sales_update.md`, но
  сохранена прежняя понравившаяся структура первого экрана: слева продуктовая
  подача и mockup-карточки, справа реальный блок входа в систему.
- Заголовки и типографика сделаны спокойнее и компактнее; hero теперь
  лаконично объясняет продукт для владельцев автопарков и даёт CTA на пилот/
