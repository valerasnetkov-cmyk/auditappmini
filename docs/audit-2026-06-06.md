# Audit 2026-06-06

Ежедневная сводка изменений за 2026-06-06. Файл содержит только изменения,
зафиксированные в этот день.

## AuditAvto Sales Landing

- Публичный лендинг `/` обновлён по `auditavto_landing_sales_update.md`, но
  сохранена прежняя понравившаяся структура первого экрана: слева продуктовая
  подача и mockup-карточки, справа реальный блок входа в систему.
- Заголовки и типографика сделаны спокойнее и компактнее; hero теперь
  лаконично объясняет продукт для владельцев автопарков и даёт CTA на пилот/
  демо без потери доступа к login-сценарию.
- Добавлена полная структура `боль → решение → выгода`: блоки про хаос в
  чатах и таблицах, цифровой отчёт AuditAvto, сценарий осмотра, фотофиксацию
  дефектов, пробег и ДТП, безопасность и роли, выгоды владельца, пилотный
  запуск, FAQ и финальный CTA.
- Важный текст реализован HTML-разметкой, изображения используются как
  иллюстрации через `next/image`, а визуальная система страницы вынесена в
  `web/src/app/landing.module.css` без inline-style в TSX.
- SEO metadata для `/` обновлены на публичное позиционирование:
  `AuditAvto — цифровой контроль автопарка и фотофиксация осмотров`.
- Browser/IAB и Playwright QA проверили `/`, наличие login-блока в hero,
  CTA href, `/login?next=/dashboard`,
  отсутствие горизонтального overflow на desktop/mobile и загрузку изображений;
  evidence screenshots сохранены в `release-evidence/landing-concise/`.
- Verification: `npm run lint` в `web` (0 errors, 3 pre-existing warnings про
  `<img>` в inspection-компонентах), `npm run build` в `web` (успешно, routes
  включают `/`, `/dashboard` и `/login`).
- Landing polish: supplied `icon.png` подключён как metadata icon и Next app
  icon (`/icon.png` появился в build routes), а веса/размеры заголовков
  уменьшены для более спокойной визуальной иерархии. Playwright подтвердил
  ссылки на `/auditavto/icon.png`.
- Design refinement: hero дополнительно уплотнён визуально — H1 стал легче,
  тени карточек мягче, первый экран на desktop сохранил преимущества рядом с
  продуктовой подачей, а на mobile блок `Вход в систему` поднят выше mockup-
  карточек и появляется сразу после вводного блока/CTA.
