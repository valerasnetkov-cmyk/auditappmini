# Audit report - 2026-06-25

## Pilot request admin actions

- В модальном окне заявки на пилот добавлена кнопка “Отклонить”; действие
  запрашивает причину отказа и сразу сохраняет статус `rejected`.
- Добавлена кнопка “Добавить компанию”, которая переводит администратора на
  вкладку компаний с query-контекстом заявки и предзаполненным названием
  компании.
- Страница компаний подхватывает переход `create=company` и переносит тариф
  заявки в форму лимитов как стартовый план.

## Landing pilot modal tariff routing

- На лендинге CTA пилота, демонстрации и карточек тарифов больше не ведут на
  `mailto`; все такие действия открывают модальное окно заявки.
- В модальное окно добавлен обязательный выбор тарифа: `Пилот`, `Стандарт` или
  `Enterprise`.
- При открытии заявки из карточки тарифа выбранный тариф подставляется
  автоматически и сохраняется в `pilot_requests.preferred_plan_code`.
- Resource-admin показывает выбранный тариф в списке и карточке заявки, а
  conversion preview использует его как тариф будущей компании с fallback на
  `pilot`.

## Secure public PDF report links

- QR в PDF-отчётах оставлен как публичная проверка подлинности, но базовый URL
  теперь строится через `WEB_APP_URL`; старые `WEB_URL` / `PUBLIC_WEB_URL`
  остаются fallback только для совместимости.
- Production guard запрещает пустой, non-HTTPS, localhost/LAN/dev-host URL для
  публичных QR-ссылок отчётов.
- Добавлен `PUBLIC_REPORT_TOKEN_TTL_DAYS` с default `30`; значение `0` делает
  ссылку проверки бессрочной.
- `inspection_reports.public_pdf_enabled` добавлен как opt-in флаг: публичная
  metadata отчёта остаётся доступной по токену, но `/api/reports/public/:token.pdf`
  отдаёт PDF только после явного включения.
- Public report page показывает подтверждение целостности всегда, а кнопку PDF
  только когда backend вернул разрешённый `pdf_url`.

## Resource-admin photo evidence modal

- В карточку компании resource-admin добавлен блок последних фото компании:
  миниатюры открываются в модальном окне внутри панели, а не сразу в новой
  вкладке.
- Модальное окно лениво запрашивает backend-генерацию `watermark.webp`,
  показывает водяную версию фото и содержит отдельную ссылку “Открыть полную
  версию” в новой вкладке.
- Resource-admin получил доступ к защищённым upload-файлам для фото компаний;
  tenant-пользователи по-прежнему ограничены своей компанией.
- Строки водяного знака вынесены в общий helper и теперь включают координаты из
  `captured_lat` / `captured_lng`, fallback на `geo`, иначе “недоступны”.
- Web-загрузка фото осмотра/дефекта теперь пытается получить browser
  geolocation и отправляет `captured_lat` / `captured_lng` вместе с `geo`.
