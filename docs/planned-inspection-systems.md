# Planned Inspection: Internal and Technical Systems

## Назначение

Файл описывает дополнительные блоки планового осмотра, которые выходят за рамки внешней фотофиксации кузова.

Плановый осмотр должен фиксировать не только внешние дефекты, но и техническое состояние доступных для проверки систем:

```txt
ходовая часть
тормозная система
электрика
световые приборы
```

Проект по-прежнему не является системой ремонта. Инспектор не ставит диагноз ремонта и не назначает исполнителя. Его задача — зафиксировать факт, фото, комментарий и состояние по чек-листу.

---

## Принцип

Плановый осмотр состоит из двух уровней:

```txt
1. Обязательная внешняя фотофиксация автомобиля.
2. Расширенный технический чек-лист по внутренним/техническим системам.
```

Детальные фото технических узлов требуются:

- если в пункте чек-листа выбран ответ `НЕТ`;
- если инспектор видит дефект или отклонение;
- если регламент компании требует фото конкретного узла;
- если нужно подтвердить спорный момент.

Если доступ к узлу невозможен без разборки, инспектор выбирает статус `not_available` или оставляет комментарий о невозможности проверки.

---

## Блоки планового осмотра

### 1. Внешнее состояние

Проверяется состояние кузова и внешних элементов.

Примеры пунктов:

- фронтальная часть без повреждений;
- левый борт без повреждений;
- правый борт без повреждений;
- задняя часть без повреждений;
- стекла и зеркала без повреждений;
- номерной знак читаемый;
- кузовные элементы закреплены корректно.

Фото:

- `front`;
- `left_side`;
- `right_side`;
- `rear`;
- `overall`;
- `defect`, если есть отклонение.

---

### 2. Ходовая часть

Проверяются доступные элементы ходовой части без разборки.

Примеры пунктов:

- нет визуальных повреждений элементов подвески;
- нет видимых течей амортизаторов;
- нет повреждений пыльников и защитных элементов;
- колёса и шины без критичных повреждений;
- нет очевидных перекосов или просадок;
- крепления колёс визуально в норме.

Фото:

- `undercarriage` или `component_detail`, если требуется регламентом;
- `defect`, если выбран ответ `НЕТ`.

---

### 3. Тормозная система

Проверяются доступные элементы тормозной системы и признаки неисправностей.

Примеры пунктов:

- нет видимых подтёков тормозной жидкости;
- тормозные шланги/трубки без видимых повреждений;
- тормозные диски/барабаны без очевидных повреждений, если доступны для осмотра;
- стояночный тормоз работает, если проверка предусмотрена регламентом;
- нет предупреждений тормозной системы на панели приборов.

Фото:

- `brake_system` или `component_detail`, если узел доступен и требуется фиксация;
- `defect`, если выбран ответ `НЕТ`.

---

### 4. Электрика

Проверяются доступные электрические элементы и признаки неисправностей.

Примеры пунктов:

- аккумулятор и клеммы без видимых повреждений/окисления, если доступны;
- видимая проводка без повреждений;
- приборная панель без критичных ошибок, если проверка предусмотрена;
- стеклоочистители и омыватель работают, если включены в регламент;
- звуковой сигнал работает, если включён в регламент.

Фото:

- `electrical` или `component_detail`, если есть отклонение;
- `defect`, если выбран ответ `НЕТ`.

---

### 5. Световые приборы

Свет относится к блоку электрики, но для удобства должен быть отдельной секцией в чек-листе.

Примеры пунктов:

- ближний свет работает;
- дальний свет работает;
- габаритные огни работают;
- стоп-сигналы работают;
- указатели поворота работают;
- аварийная сигнализация работает;
- фонари заднего хода работают;
- подсветка номера работает;
- противотуманные фары/фонари работают, если установлены.

Фото:

- `lighting`, если требуется регламентом или есть отклонение;
- `defect`, если выбран ответ `НЕТ`.

---

## Статусы пунктов планового осмотра

Для планового осмотра одного `ДА / НЕТ` может быть недостаточно. Рекомендуется поддержать статусы:

```txt
yes            -> проверено, отклонений нет
no             -> есть дефект / отклонение
not_applicable -> не применимо для этой техники
not_available  -> невозможно проверить без доступа или разборки
```

Для MVP можно оставить `ДА / НЕТ`, но предусмотреть в модели возможность расширения.

---

## Правила дефектов

Любой ответ `НЕТ` в техническом блоке создаёт дефект.

Дефект должен содержать:

```txt
component_area     -> undercarriage / brake_system / electrical / lighting / exterior / other
title              -> краткое название дефекта
description        -> комментарий инспектора
photo              -> обязательное фото, если дефект видимый
severity           -> low / medium / high, если используется
```

Если дефект нельзя сфотографировать напрямую, инспектор должен указать причину и добавить доступное подтверждающее фото или комментарий.

---

## Что не делать

- Не превращать плановый осмотр в ремонтную карту.
- Не требовать разборки узлов.
- Не давать инспектору задачу ставить технический диагноз, если он только фиксирует состояние.
- Не завершать пункт `НЕТ` без комментария и, где возможно, без фото.
- Не смешивать дефекты кузова, ходовой, тормозов и электрики в одной общей категории.
