## Odometer OCR provider

- Добавлен backend provider layer для OCR одометра:
  `OCR_ODOMETER_PROVIDER=mock` сохраняет manual-confirmation placeholder, а
  `OCR_ODOMETER_PROVIDER=tesseract-cli` запускает системный Tesseract CLI на
  backend host.
- `/api/odometer/recognize` стал async route: принимает прежнее multipart поле
  `photo`, прогоняет изображение через `sharp`-предобработку для Tesseract и
  возвращает `raw_value`, `normalized_value`, `confidence`, `candidates`,
  `provider`, `requires_manual_confirmation`, `photo_url` и `recognized_at`.
- Принцип доказательного осмотра не изменён: OCR только подсказывает значение,
  финальный пробег сохраняется прежним `POST /api/inspections/:id/odometer`
  после подтверждения инспектором. OCR госномера оставлен placeholder-ручкой,
  потому что для production-quality номера нужен отдельный ANPR/LPR provider.
- Production env и release docs обновлены: `OCR_ODOMETER_PROVIDER`,
  `TESSERACT_CMD`, `TESSERACT_TIMEOUT_MS`; `doctor:production` проверяет
  `tesseract --version` только когда включён `tesseract-cli`.
- Добавлены unit-тесты парсинга/нормализации OCR текста и targeted
  `smoke:ocr`, который проверяет mock-provider path без установленного
  Tesseract и фиксацию OCR usage.
- Добавлен ручной staging smoke `npm --prefix backend run smoke:ocr:tesseract`:
  локально он безопасно завершается `skipped`, если provider не
  `tesseract-cli` или системный `tesseract` недоступен; на pilot/staging
  проверяет реальный endpoint и фиксацию OCR usage.
