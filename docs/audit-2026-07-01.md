# Daily audit 2026-07-01

## Scope

- Continued the operational implementation after docs integration, alert
  dry-run, worker foundation, storage abstraction and resource-admin support
  notes.
- Focused on closing a small resource-admin health gap: worker status evidence
  existed, but the health item still described the old static `not_configured`
  placeholder.

## Changes

- Resource-admin health center now maps worker evidence dynamically:
  `not_configured`, `idle`, `running` and `degraded`.
- `degraded` worker status is surfaced as high severity when failed jobs or
  heartbeat errors are present.
- `running` and `idle` worker foundation states are surfaced as OK instead of
  suggesting that workers are still missing.
- `smoke:workers` now verifies that heartbeat evidence is reflected in the
  resource-admin worker health item.
- `docs/resource-admin-processes.md` now documents support notes as implemented
  and worker health as a dynamic status set.
- Fixed pilot trial countdown drift: legacy `company_billing.trial_until`
  values are capped to `company.created_at + 30 days` for tenant usage,
  operational write policy and the billing status job.
- Added startup repair for stored overlong trial dates and web usage refresh on
  focus/visibility/hourly interval so open pages do not keep stale day counters.
- Implemented optional service-admin Telegram bot for internal resource-admin
  signals: pilot requests, company/payment events, subscription alerts, PDF
  failures and critical defects. Tenant roles are not connected to Telegram,
  and messages exclude photos, PDFs, setup links and secrets.
- Fixed vehicle primary-photo display for inspection-sourced covers: stale
  rear/odometer/defect saved covers are normalized to the latest common-plan
  inspection photo in vehicle detail and list responses, while manual uploads
  remain untouched.
- Follow-up correction: the live inspection requirement uses `photo_type =
  'overview'` for “Фото общего плана”; backend primary-photo filtering now
  accepts `overview` plus legacy `overall`, while still excluding `rear`,
  odometer, defect and `accident_overview_*` photos.
- Added a compact vehicle thumbnail column to the web `Техника` table using the
  normalized primary photo.
- Added `docs/deploy-vps.md` as the production VPS deployment runbook for
  `/opt/auditappmini`, fixing the canonical deploy command
  `./scripts/deploy-from-master.sh`, 502 diagnostics, PM2 web process shape and
  stable `NEXT_SERVER_ACTIONS_ENCRYPTION_KEY` requirement.
- Added `scripts/templates/ecosystem.config.vps.cjs` as a non-binding VPS PM2
  template that runs `audit-web` directly through Next.js on
  `127.0.0.1:3002`.
