## 6. Рекомендации к пилоту

1. **Pre-pilot:** создать `backend/.env.production`, `web/.env.production` и
   EAS environment variables на реальном pilot/staging-хосте, прогнать
   `npm run doctor:production`. Должен вернуть `ok: true`.
2. **Pre-release:** `npm run backup:local` → `npm run backup:verify` →
   `npm run release:evidence` → приложить манифест к release-нотам.
3. **First start:** `npm run release:first-start` → проверить
   `pm2 status` → `pm2 logs audit-backend` (первый запуск).
4. **Post-start UAT:** ручной прогон admin/owner/manager/inspector-сценариев
   на чистом staging.
5. **Cleanup-волна (опционально):** после успешного пилота — выполнить шаги
   2.1–2.6 (удалить артефакты, перенести БД в `backend/data/`, све `.md` в `docs/`).
6. **Архитектурный долг:** начать с Epic 3.1 (`sql.js` → `better-sqlite3`) —
   это снизит RAM-нагрузку и упростит миграцию при росте данных.
