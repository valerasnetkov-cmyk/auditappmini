# Audit report - 2026-06-28

## Mobile OCR recognition flow

- Проверен мобильный OCR-поток: backend endpoint `/api/odometer/recognize`
  работал, но в мобильном шаге “Одометр” не было кнопки камеры/OCR, поэтому
  пользователь мог только вводить пробег вручную.
- В мобильный шаг одометра добавлена кнопка “Распознать по фото”; после снимка
  приложение вызывает OCR, подставляет найденное значение в поле пробега и
  просит инспектора проверить/подтвердить результат.
- OCR номера техники перестал быть backend-заглушкой: при включенном
  `OCR_ODOMETER_PROVIDER=tesseract-cli` backend запускает Tesseract, извлекает
  кандидаты госномера и нормализует латинские look-alike символы в кириллицу.
- В `mock`-режиме OCR по-прежнему возвращает ручное подтверждение без
  распознанного значения; для production нужен `OCR_ODOMETER_PROVIDER=tesseract-cli`
  и доступный `TESSERACT_CMD`.
