# Odometer Recognition

## Назначение

Файл описывает фиксацию одометра, распознавание километража по фото и ручное подтверждение инспектором.

---

## Главный принцип

Распознавание километража не является финальным источником данных.

Правильный сценарий:

```txt
Фото одометра -> OCR -> предполагаемый километраж -> подтверждение или ручное исправление инспектором -> запись в карточку осмотра
```

Ручной ввод километража должен быть доступен всегда.

---

## Когда фото одометра обязательно

| Тип осмотра | Фото одометра | Километраж |
|---|---:|---:|
| Быстрый | обязательно | обязательно |
| Плановый | обязательно | обязательно |
| ДТП | если доступно | если доступно или причина недоступности |

---

## UI-сценарий mobile

1. Инспектор доходит до шага `Одометр`.
2. Приложение открывает камеру.
3. Инспектор фотографирует одометр.
4. Система пытается распознать километраж.
5. На экране показывается результат.
6. Инспектор нажимает `Подтвердить` или исправляет значение вручную.
7. Подтверждённый километраж сохраняется в карточку осмотра.


## Валидация

Километраж:

- только целое число;
- единица измерения по умолчанию — `km`;
- не может быть отрицательным;
- не должен быть меньше предыдущего подтверждённого значения без предупреждения;
- при подозрительном значении осмотр не блокируется, но получает флаг `requires_review`.

Пример подозрительного значения:

```txt
предыдущий пробег: 180000 км
новый пробег: 120000 км
```

## Backend API

### POST /api/odometer/recognize

Назначение: принять фото одометра и вернуть распознанный километраж.

Ответ:

```json
{
  "rawText": "128450",
  "recognizedValue": 128450,
  "unit": "km",
  "confidence": 0.86,
  "candidates": [
    { "value": 128450, "confidence": 0.86 }
  ]
}
```

### POST /api/inspections/:id/odometer

Назначение: сохранить подтверждённый километраж.

Запрос:

```json
{
  "odometerValue": 128450,
  "unit": "km",
  "photoId": "photo-id",
  "recognitionId": "recognition-id"
}
```

## Что не делать

- Не завершать осмотр только на основании OCR без подтверждения инспектора.
- Не скрывать ручной ввод.
- Не сохранять распознанное значение как финальное без подтверждения.
- Не удалять фото одометра после распознавания.
- Не хранить фото одометра в публичной папке репозитория.

## Километры и мили

Распознавание одометра работает с числом, а единица измерения берётся из настроек компании:

```txt
company.distance_unit = km | mi
```

OCR не должен самостоятельно менять единицу измерения компании. Если на фото одометра видна маркировка `mi`, а компания использует `km`, система показывает предупреждение, но финальное значение всё равно подтверждает инспектор.

Сохранять нужно:

```txt
odometer_value
odometer_unit
odometer_value_km
```

`odometer_value_km` используется для проверки уменьшения пробега и аналитики.
