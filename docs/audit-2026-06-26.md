# Audit report - 2026-06-26

## Mobile inspection flow controls

  осмотром.

## Vehicle number repair tenant scope

- Исправлен startup-repair номеров техники: проверка дублей теперь учитывает
  `company_id`, а не весь общий список `vehicles`.
- Валидный номер `К002МК65`, исправленный вручную в карточке авто, больше не
  должен превращаться обратно в `К002МК265` из-за совпадения номера в другой
  компании или демо-контуре.
- Добавлен unit regression test на одинаковый валидный номер в двух разных
  компаниях после повторного `initDatabase()`.

## Inspection photo modal access fix

- Ошибочное решение с доступом resource-admin к файлам компаний откатано:
  `/uploads/*` снова проверяет `company_id` текущего tenant-пользователя.
- Из resource-admin карточки компании удалён блок последних фото, API-метод
  генерации watermark для фото компании и поле `recentPhotos` в деталях
  компании сервиса.
- Модальное окно просмотра фото реализовано в корректном tenant-контексте: на
  странице осмотра, в блоке “Обязательные фото осмотра”, миниатюра открывает
  фото внутри интерфейса.
- При открытии фото tenant-only endpoint `/api/photos/:id/watermark` лениво
  формирует `watermark.webp` с компанией, техникой, ID/типом осмотра, временем
  и координатами.
- В модальном окне осмотра и в ссылке “Открыть полную версию” используется
  watermarked-файл; cross-tenant генерация watermark возвращает `404`.
