  hero по принятому макету, продуктовые карточки, реальная форма входа,

  hero лендинга, сохраняя login/MFA flow, `data-testid` поля и role-based
  redirect для admin/non-admin пользователей.
- Brand/photo assets скопированы в `web/public/auditavto/`; desktop/mobile QA
  screenshots сохранены в `release-evidence/landing/`.
- Verification: `npm run lint` в `web` (0 errors, 3 pre-existing warnings про
  `<img>` в inspection-компонентах), `npm run build` в `web` (успешно, routes
  включают `/` и `/dashboard`), Browser/IAB + Playwright проверили публичный
  лендинг, submit ошибки формы и unauthenticated `/dashboard` redirect.

## Remaining `audit.md` Follow-ups

- Старый `audit.md` использован как исторический вход для сверки рисков; сам
  файл остаётся untracked и не входит в проектную документацию.
- `GET /api/vehicles`: устранён N+1 для `lastInspection` и `defectsCount`.
  Список техники теперь получает последний осмотр и количество дефектов одним
  SQL-запросом через aggregated subqueries/`LEFT JOIN`, сохраняя response shape
  и текущую pagination.
- MFA lifecycle: добавлен явный `POST /api/users/:id/mfa/enable`; старый
  `POST /api/users/:id/mfa/verify` оставлен backwards-compatible alias. Web
  MFA UI и API client переведены на `/mfa/enable`.
- MFA disable: добавлен `POST /api/users/:id/mfa/disable` для пользователя в
  той же компании. Операция требует пароль текущего пользователя и валидный
  TOTP target user, если MFA у него уже включён.
- Login challenge flow не изменялся: `/api/auth/login` возвращает `mfaToken`,
  а `/api/auth/mfa/verify` завершает вход.
- `speakeasy` в web остаётся только devDependency для `web/e2e/tests/*`; runtime
  web bundle не должен импортировать этот пакет.
- JWT refresh/revocation не внедрялся в этом chunk. Актуальный оставшийся риск:
  7-дневный JWT без server-side revocation list; следующий security epic должен
  проектировать server-side sessions + refresh-cookie flow.
- Runtime artifacts (`.tmp-*`, `backend/data`, `backend/uploads`,
  `backend/backups`) считаются ignored operational storage; cleanup выполняется
  локально/операционно, а не коммитится как кодовое изменение.
- `backend/.env.example` остаётся dev-friendly; `ADMIN_PASSWORD=admin123`
  допустим только локально. Production doctor/config блокируют demo password.
