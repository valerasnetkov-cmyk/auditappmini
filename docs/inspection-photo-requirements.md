# Inspection Photo Requirements

## Назначение

Файл фиксирует обязательные типы фото для разных сценариев осмотра.

---

## Типы фото

```txt
front                 -> фронтальная часть автомобиля
left_side             -> левый борт
right_side            -> правый борт
rear                  -> задняя часть автомобиля
overall               -> общий план
odometer              -> одометр
number_plate          -> госномер / номерной знак
interior              -> салон, если требуется
undercarriage         -> ходовая часть / доступные элементы снизу
brake_system          -> тормозная система
electrical            -> электрика
lighting              -> световые приборы
component_detail      -> детальное фото технического узла
defect                -> дефект по чек-листу
accident_overall      -> общий план повреждения при ДТП
accident_damage_close -> крупный план повреждённого участка
other                 -> дополнительное фото
```

---

Минимально обязательные фото:

| Тип фото | Обязательное | Комментарий |
|---|---:|---|
| `front` | да | Фронтальная часть автомобиля |
| `left_side` | да | Левый борт |
| `right_side` | да | Правый борт |
| `rear` | да | Задняя часть автомобиля |
| `overall` | да | Общий план автомобиля |
| `odometer` | да | Фото одометра для фиксации километража |
| `defect` | условно | Обязательно при ответе `НЕТ` |

## Осмотр ДТП

Минимально обязательные фото:

| Тип фото | Обязательное | Комментарий |
|---|---:|---|
| `overall` | да | Общий план автомобиля |
| `front` | да | Фронтальная часть автомобиля |
| `left_side` | да | Левый борт |
| `right_side` | да | Правый борт |
| `rear` | да | Задняя часть автомобиля |
| `accident_overall` | да | Общий план повреждённой зоны |
| `accident_damage_close` | да | Крупный план каждого повреждённого участка |
| `odometer` | если доступно | Если доступ невозможен, указывается причина |

Дополнительно к фото для ДТП обязательны не фотографические данные:

```txt
accident_occurred_at -> дата и время ДТП
accident_place_text  -> место ДТП текстом
accident_latitude    -> координата, если доступна
accident_longitude   -> координата, если доступна
```

---

## Правила для ДТП

- Если повреждение одно, нужен минимум один `accident_damage_close`.
- Если повреждений несколько, каждое повреждение фиксируется отдельным фото `accident_damage_close`.
- Общий план повреждения нужен для контекста, но не заменяет close-up.
- Close-up должен показывать повреждение достаточно крупно, чтобы его можно было проверить в web-интерфейсе.
- Место и время ДТП обязательны для завершения ДТП-осмотра.
- Время осмотра и геоданные фото не заменяют место и время ДТП.

## Проверка завершения осмотра

Перед завершением backend проверяет обязательные фото по `inspection_type`.

Если обязательного фото нет, backend возвращает ошибку со списком недостающих типов.

Пример:

```json
{
  "error": "missing_required_photos",
  "missing": ["front", "odometer"]
}
```

## Плановый осмотр: технические фото

Для планового осмотра внешние фото обязательны всегда, а технические фото обязательны по условию.

Условие обязательности технического фото:

```txt
если section = undercarriage / brake_system / electrical / lighting
и result = false
то требуется фото defect или component_detail
```

Рекомендуемые типы фото:

| Секция | Основной тип фото | Когда требуется |
|---|---|---|
| Ходовая часть | `undercarriage` | при видимом дефекте, спорном моменте или регламенте |
| Тормозная система | `brake_system` | при подтёках, повреждении, предупреждениях или регламенте |
| Электрика | `electrical` | при повреждении проводки, клемм, ошибках или регламенте |
| Свет | `lighting` | при неисправности светового прибора или регламенте |
| Любой технический узел | `component_detail` | когда нужно показать конкретный узел крупно |

Если фото сделать невозможно из-за отсутствия доступа, инспектор должен оставить комментарий с причиной.
