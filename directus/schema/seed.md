# Directus seed and roles

Этот файл описывает стартовые значения и роли для ручной настройки Directus Studio в модели Plan B.

## Роли

- `Resource Admin` - единственная интерактивная роль в Directus Studio. Администратор ресурса управляет SaaS-проектом: компаниями, владельцами компаний, тарифами, лимитами, feature flags и служебными заметками.
- `Service Token` - технический доступ custom backend к Directus REST API для provisioning компаний и владельцев.

Владельцы компаний, менеджеры и инспекторы не создаются как пользователи Directus Studio. Владелец компании хранится как запись `company_owners`, затем через provisioning sync создается локальный пользователь `owner` и управляет менеджерами/инспекторами только в пользовательской панели своей компании.

## Рекомендуемые справочники

### companies.type

- `insurance` - страховая компания.
- `leasing` - лизинговая компания.
- `fleet` - владелец автопарка.
- `other` - другой тип клиента.

### common status

- `active` - компания, владелец или тариф активны.
- `inactive` - отключено, скрыто или временно не используется.

### Базовые тарифы plans.code

- `pilot` - пилотный запуск.
- `standard` - рабочий тариф.
- `enterprise` - индивидуальный тариф.

Эти тарифы можно внести автоматически:

```bash
npm run directus:seed:saas
```

Скрипт создает:

- `pilot`: 10 единиц техники, 3 пользователя, 512 МБ, OCR и ДТП включены, аналитика/API отключены.
- `standard`: 100 единиц техники, 15 пользователей, 5120 МБ, OCR/ДТП/аналитика включены, API отключен.
- `enterprise`: без заданных лимитов, OCR/ДТП/аналитика/API включены.

`company_limits` создаются только для компаний, которые уже есть в Directus. Если компаний нет, лимиты не создаются, чтобы не появлялись сиротские записи. Если у существующей компании пустой `plan_code`, seed проставляет `pilot` и создает лимит по пилотному тарифу. Уже созданные лимиты не перезаписываются без флага `--force`.

### company_limits flags

- `ocr_enabled` - доступно распознавание номера/одометра.
- `accident_module_enabled` - доступен поток ДТП-фиксации.
- `analytics_enabled` - доступна аналитика в пользовательской панели.
- `api_access_enabled` - разрешен внешний API-доступ, если он будет включен в будущем.

## Первые действия

1. Создать администратора при первом запуске через `.env`.
2. Создать роль `Resource Admin` и service token для custom backend.
3. Создать тарифы в `plans`.
4. Создать компанию в `companies` и владельца в `company_owners`.
5. При необходимости заполнить `company_limits`.
6. Добавить token в `backend/.env` как `DIRECTUS_TOKEN`.
7. Запустить backend provisioning sync и передать владельцу одноразовую setup-ссылку для установки пароля в пользовательской панели.
8. Проверить, что web/mobile не получают прямой доступ к Directus и не показывают CMS компаниям.
