🔧 1. Внутренний механик

Как работает:

ему назначают задачи

🏢 2. Подрядчик (внешняя компания)

Как работает:

задача уходит наружу

Что меняется:

у подрядчика может не быть доступа в систему
факт выполнения фиксирует кто-то внутри

Что нужно:

поле "подрядчик" (просто текст)
загрузка фото "после"

Сложность:
🟢 низкая (если не давать доступ подрядчику)

🚗 3. Водитель

Как работает:

сам устраняет мелкие дефекты (мойка, лампочка и т.д.)

Что нужно:

доступ водителю
мобильное приложение для него

Сложность:
🔴 высокая

Почему:

появляется ещё одна роль
риск “сам себе закрыл задачу”
падает достоверность
🔀 4. Смешанная модель

Как работает:

часть задач → механик

часть → водитель

Что нужно:

разная логика выполнения

Сложность:
🔴 максимальная

🧱 1. Структура БД (готовый SQL)

Это можно просто вставить в Supabase → SQL Editor.

-- USERS (расширение профиля)
create table users (
    id uuid primary key references auth.users(id),
    name text,
    role text check (role in ('inspector', 'manager')),
    created_at timestamp default now()
);

-- VEHICLES
create table vehicles (
    id uuid primary key default gen_random_uuid(),
    number text not null,
    name text,
    status text default 'active',
    created_at timestamp default now()
);

-- INSPECTIONS
create table inspections (
    id uuid primary key default gen_random_uuid(),
    vehicle_id uuid references vehicles(id) on delete cascade,
    inspector_id uuid references users(id),
    type text check (type in ('quick', 'full')),
    created_at timestamp default now(),
    completed boolean default false
);

-- CHECKLIST ITEMS
create table checklist_items (
    id uuid primary key default gen_random_uuid(),
    inspection_id uuid references inspections(id) on delete cascade,
    title text,
    result boolean,
    comment text
);

-- DEFECTS
create table defects (
    id uuid primary key default gen_random_uuid(),
    inspection_id uuid references inspections(id) on delete cascade,
    title text,
    created_at timestamp default now()
);

-- PHOTOS
create table photos (
    id uuid primary key default gen_random_uuid(),
    inspection_id uuid references inspections(id) on delete cascade,
    defect_id uuid references defects(id) on delete set null,
    url text,
    geo text,
    created_at timestamp default now()
);
